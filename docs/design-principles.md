# Design Principles

Lumen is designed for low cognitive load: calm surfaces, clear explanations, and confirmation before meaningful actions.

## Principles

- Make the current home state easy to scan.
- Show why a suggestion exists before asking for action.
- Keep device control available, but do not let it dominate the app.
- Prefer direct labels over technical automations language.
- Avoid surprise execution; meaningful actions need confirmation.
- Keep preview mode useful enough for demos and testing without hardware.

## Tone

Use calm, practical language. Avoid clinical claims unless they are backed by research, testing, and explicit product positioning.

## Interaction Standard

Every scene path should answer:

1. What will change?
2. Why is Lumen suggesting it?
3. What happens if the user says no?
4. Where would an error appear?
